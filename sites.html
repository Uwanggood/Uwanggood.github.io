새 사이트 이름 규칙

  1. slug는 소문자와 하이픈만 씁니다. 예: path-doctor
  2. GitHub Pages 주소는 이 slug가 보이게 만듭니다. 예: /path-doctor/
  3. 도메인을 사면 이 페이지와 sites.json에 같이 기록합니다.
  4. 사이트를 만들 때마다 README의 표, sitemap, 이 목록을 같이 갱신합니다.

앞으로 이렇게 쌓으면 됩니다

사이트가 20개가 되어도 주소와 도메인 상태를 이름순으로 찾을 수 있게 같은 형식으로 기록합니다.
