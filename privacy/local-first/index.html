Local-first Desktop Apps Privacy Policy

This policy applies to Uwanggood desktop apps that work primarily on your device and do not require a Uwanggood account server.

Last updated: 2026-06-28

Applies to: Bucket Studio and Path Doctor.

Short version.

These apps are designed to process your work locally. They do not sell personal information, do not show ads, and do not use advertising trackers. They do not send your cloud credentials, storage contents, local files, or diagnostic history to a Uwanggood application server.

1. Apps covered by this policy

This policy covers local-first desktop apps published by Uwanggood, including:

  • Bucket Studio: a desktop browser for S3-compatible storage such as AWS S3, Cloudflare R2, Wasabi, MinIO, and compatible private endpoints.
  • Path Doctor: a desktop network diagnostic tool for checking DNS, ping, HTTP response, traceroute, VPN, and proxy-related clues.

If a future app adds accounts, hosted cloud storage, advertising analytics, or server-side user profiles, it should use a separate policy URL instead of this local-first policy.

2. Information processed on your device

Depending on the app and the features you choose, the app may process information locally on your device.

  • Bucket Studio may process storage provider names, endpoint URLs, bucket names, object keys, prefixes, transfer status, selected local file paths, and provider credentials you enter.
  • Path Doctor may process target host names, URLs, IP addresses, DNS results, ping results, HTTP response data, traceroute output, proxy/VPN clues, app settings, and recent diagnostic history.

This local processing is used to provide the app features you request.

3. Credentials and sensitive local data

Bucket Studio is designed to keep storage credentials on your device. Where available, credentials are stored using platform secure storage. Bucket Studio does not send access keys, secret keys, session tokens, bucket names, object paths, presigned URLs, or provider endpoints to a Uwanggood application server.

Path Doctor stores diagnostic settings and recent history locally so you can review previous checks. It does not upload diagnostic history to a Uwanggood application server.

4. Network requests made by the apps

The apps may make network requests when needed for the feature you choose:

  • Bucket Studio connects directly from your device to the storage endpoint you configure.
  • Path Doctor sends diagnostic requests to the host, URL, or IP address you choose to test.
  • The apps may check GitHub Pages or GitHub Releases for public update information.
  • If you download, buy, donate, or install through Microsoft Store, GitHub, itch.io, Ko-fi, or another platform, that platform processes information under its own terms and privacy policy.

5. Diagnostics and crash reporting

The apps do not use advertising analytics. Bucket Studio release builds may use limited crash diagnostics to help fix serious app errors. When crash diagnostics are enabled, they are configured to avoid screenshots, request bodies, failed request capture, HTTP breadcrumbs, and default personally identifiable information.

Diagnostic events are scrubbed before sending. Credentials, signatures, tokens, cookies, presigned URL query strings, bucket names, object keys, endpoints, and local paths should be redacted.

If a future release materially expands diagnostics or analytics, the relevant app page or policy URL should be updated before release.

6. Data sharing

Uwanggood does not sell personal information. Uwanggood does not share your cloud credentials, storage contents, local files, or diagnostic history with advertisers or data brokers.

Data may be processed by third-party platforms only when you choose to use those platforms, such as downloading a release from GitHub, installing through Microsoft Store, donating through Ko-fi, or buying through itch.io.

7. Data deletion

You can remove app data by deleting saved profiles, clearing app history where the app provides that option, uninstalling the app, or deleting the app's local data folder from your operating system profile.

For Bucket Studio, removing a provider profile removes the locally stored provider entry and associated local credentials from Bucket Studio. It does not delete objects from your storage provider unless you explicitly run a delete operation in the app.

8. Children

These apps are productivity and utility tools. They are not directed to children.

9. Changes to this policy

This policy may be updated when app features, distribution platforms, diagnostics, or legal requirements change. The latest version will be published at this URL.

10. Contact

For privacy or security questions, use the relevant official project page or GitHub repository listed on the Uwanggood site registry. Do not include cloud credentials, private endpoints, presigned URLs, object names, or sensitive logs in public reports.
