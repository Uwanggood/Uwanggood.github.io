DNS guide

DNS는 정상인데 웹사이트가 안 열릴 때

DNS가 성공했다는 것은 “주소를 찾았다”는 뜻입니다. 실제 웹 접속은 그 다음 단계에서 실패할 수 있습니다.

DNS 성공 이후에도 실패할 수 있는 이유

  • 서버가 HTTP 또는 HTTPS 요청에 응답하지 않음
  • 방화벽이나 보안 프로그램이 연결을 막음
  • VPN, 프록시, 회사망 정책이 특정 도메인을 차단함
  • TLS 인증서나 SNI 단계에서 문제가 생김
  • 라우팅 경로 중간에서 패킷 손실이 발생함
핵심

DNS는 전화번호부에서 번호를 찾는 단계에 가깝습니다. 전화를 실제로 걸었을 때 상대가 받는지는 HTTP 응답을 봐야 합니다.

확인 순서

  1. 같은 주소를 브라우저와 진단 도구에서 모두 확인합니다.
  2. HTTP 상태 코드와 응답 시간을 봅니다.
  3. ping 손실과 route가 마지막까지 이어지는지 확인합니다.
  4. VPN, 프록시, 보안 프로그램을 잠시 끄고 비교합니다.
  5. 다른 네트워크에서 같은 사이트가 열리는지 확인합니다.
