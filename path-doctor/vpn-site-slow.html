VPN troubleshooting

VPN을 켜면 특정 사이트만 느릴 때 확인 순서

VPN은 보안을 위해 필요하지만, DNS와 라우팅을 바꾸기 때문에 특정 사이트만 느려지는 일이 생길 수 있습니다.

1. VPN을 끈 상태와 켠 상태를 같은 대상에서 비교

같은 사이트를 VPN off/on 상태에서 각각 진단하면 DNS 주소, HTTP 응답 시간, route hop이 어떻게 달라지는지 볼 수 있습니다.

2. DNS가 회사망 또는 VPN DNS로 바뀌었는지 확인

VPN 연결 후 내부 DNS가 사용되면 일부 외부 사이트의 위치가 다르게 해석될 수 있습니다. 지역 CDN을 쓰는 사이트에서 특히 체감됩니다.

3. 프록시나 보안 검사 장비를 의심

회사 VPN은 웹 트래픽을 보안 장비로 우회시킬 수 있습니다. 특정 도메인만 검사 대상이면 그 사이트만 느려집니다.

4. route가 멀리 돌아가는지 확인

VPN 서버 위치가 멀거나, 회사망을 한 번 거쳐 외부로 나가는 구조라면 가까운 사이트도 지연이 늘어날 수 있습니다.

비교가 핵심입니다

한 번의 진단보다 VPN off/on 결과를 나란히 비교하는 것이 원인 후보를 훨씬 빨리 줄입니다.
