Traceroute guide

tracert에서 * * * 가 나오면 인터넷 장애일까?

별표가 보인다고 바로 장애는 아닙니다. 중간 장비가 진단 신호에 답하지 않는 설정일 수 있습니다.

Windows의 tracert는 목적지까지 지나가는 중간 장비가 응답하는지 확인합니다. 그런데 많은 라우터와 방화벽은 이런 진단 신호에 일부러 답하지 않습니다.

* * * 가 나오는 흔한 이유

  • 중간 라우터가 ICMP 응답을 제한함
  • 방화벽 또는 보안 장비가 진단 신호를 낮은 우선순위로 처리함
  • VPN, 회사망, 통신사 장비가 경로를 숨김
  • 진짜로 특정 구간에서 응답이 끊김
중요한 구분

별표 이후에도 다음 hop이 계속 나오고 웹사이트가 정상 접속된다면, 중간 장비의 응답 제한일 가능성이 큽니다.

장애 가능성이 더 높은 경우

  • 별표가 나온 뒤 더 이상 경로가 이어지지 않음
  • 최종 사이트 HTTP 요청도 실패함
  • ping 손실이 높거나 응답이 불안정함
  • 다른 네트워크에서는 같은 사이트가 정상임

Path Doctor가 도와주는 부분

Path Doctor는 traceroute만 보지 않고 ping과 HTTP 응답을 함께 봅니다. 그래서 “중간 장비 응답 제한일 수 있음”과 “실제 연결 문제 가능성”을 더 쉽게 나눠 볼 수 있습니다.
